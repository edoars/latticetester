Detailed Description

template<typename Int, typename IntVec, typename BasInt, typename BasIntMat, typename RedDbl>
class LatticeTester::ParamReader< Int, IntVec, BasInt, BasIntMat, RedDbl >

Utility class used to read basic parameter fields in a configuration file.

Lines whose first non-blank character is a # are considered as comments and discarded.

Reads 2k MScal tokens into vectors B and C, starting at the ln-th line.

These represent a box \([B_i, C_i]\), \(i = 1, 2, â€¦, k\). The \(B_i, C_i\) must be given in the order \(B_1, C_1, B_2, C_2, â€¦, B_k, C_k\), each on a line of its own. Each coefficient may be given in the form described in readNumber3 above. Reads a norm from the pos-th token of the ln-th line into field.

◆ readNumber3()

template<typename Int , typename IntVec , typename BasInt , typename BasIntMat , typename RedDbl >
void LatticeTester::ParamReader< Int, IntVec, BasInt, BasIntMat, RedDbl >::readNumber3 ( Int &  r,
std::int64_t &  b,
std::int64_t &  e,
std::int64_t &  c,
unsigned int  ln,
unsigned int  pos 
)

Reads \(b\), \(e\) and \(c\), starting at the pos-th token of the ln-th line and uses them to define \(r\).

The numbers in the data file may be given in one of the two following formats:

\(\bullet\) A single integer giving the value of \(r=b\) directly on a line. In that case, one sets \(e=c=0\).
\(\bullet\) Three integers \(b\), \(e\), \(c\) on the same line, separated by at least one blank. The \(r\) value will be set as \(r=b^e+c\) if \(b>0\), and \(r= -(|b|^e+c)\) if \(b<0\). One must have \(e\ge0\). For example, \((b, e, c) = (2, 5, -1)\) will give \(r=31\), while \((b, e, c) = (-2, 5, -1)\) will give \(r=-31\).
